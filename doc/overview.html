The Avalara AvaTax Adapter provides a local API for the AvaTax Web Services.

The SDK is installed by default to C:\Program Files\Avalara\AvaTax On Demand.

The installed SDK includes:

Avalara.AvaTax.Adapter.dll (the access point for the web services),
Avalara.AvaTax.Adapter.chm (this documentation),
Avalara.AvaTax.Adapter.dll.config (an optional configuration file can be found at Program Files\Avalara\AvaTax On Demand\bin)
Except for the documentation, the above files should all be placed in the same directory.

REQUIREMENTS & REDISTRIBUTABLES

AvaTax Requirements

The AvaTax Adapter runs on the .NET Framework 2.0, 3.0 or 3.5.

Note: Installer will redirect you to URL for .NET Framework installation.

You can download the .NET Framework installer from:  http://msdn.microsoft.com/en-us/netframework/default.aspx

AvaTax Redistributables

bin\ Avalara.AvaTax.Adapter.dll
bin\ Avalara.AvaTax.Adapter.dll.config

GETTING STARTED

This documentation file provides complete coverage of all the public classes and methods that make up the AvaTax API. The classes are grouped under three primary namespaces:

Avalara.AvaTax.Adapter: contains the base classes and common classes used within the Adapter - these are supporting classes only;
Avalara.AvaTax.Adapter.AddressService: contains classes related to address and the Address web service - a good starting point is the AddressSvc class;
Avalara.AvaTax.Adapter.TaxService: contains classes related to the tax calculation web service - a good starting point is the TaxSvc class;
Avalara.AvaTax.Adapter.AvaCert2Service: contains classes related to the AvaCert web service - a good starting point is the AvaCert2Svc class.

Registering for COM

Although the AvaTax Adapter uses the Microsoft.NET Framework, it was built with full support for COM clients in mind. If you wish to use the adapter from a COM-based language, you must first register it. Included in the SDK is a batch file named register_for_com.bat. Run the file to register the Adapter assembly as a Windows type library. Alternately, it can be registered from the command line. At a command prompt, navigate to the directory containing the Adapter DLL, and enter the following: 

regasm Avalara.AvaTax.Adapter.dll /tlb /codebase

Sample Code

For a working example of the Address and Tax services, using the Adapter, check out the samples found in the SDK\Samples\ directory. There you will find two sets of samples, one written in Visual Basic 6 and the other in .NET's C#. Under each language's directory are two projects, one for AddressSvc and one for TaxSvc. One language's sample is identical to the other's, differing only in syntax and language-specific implementation.
Directory/File Description
Avalara.AvaTax.Adapter.dll.config A default configuration file. This file should be copied into the same directory as the Avalara.AvaTax.Adapter assembly. For the .NET Sample Projects, this is the bin\debug directory beneath each project directory. For the Visual Basic 6 sample projects, it should be placed in the SDK's top level bin directory.
NET\AddressSvc\ Directory containing a sample project (AddressSample.sln) written in C#. It demonstrates how to use the Avalara.AvaTax.Adapter API to access the AutoAddress+ web service.
NET\TaxSvc\ Directory containing a sample project (TaxSample.sln) written in C#. It demonstrates how to use the Avalara.AvaTax.Adapter API to access the AvaTax web service.
VB6\AddressSvc\ Directory containing a sample project (AddressSample.vbp) written in Visual Basic 6. It demonstrates how to use the Avalara.AvaTax.Adapter API to access the Address web service.
VB6\TaxSvc\ Directory containing a sample project (TaxSample.vbp) written in Visual Basic 6. It demonstrates how to use the Avalara.AvaTax.Adapter API to access the AvaTax web service.
