About CipherGen

CipherGen is a utility which encrypts text using a machine-specific key. It is necessary when adding the Account and License configuration properties to the configuration file. Using encryption keeps this sensitive data private; employing a machine-specific key ensures that the configuration file cannot be copied and used on a different machine.

It is only necessary to run this utility when there is no other option for securely storing the Account and License values. These values are considered sensitive data, and the best practice is to keep them in a password-protected database or other password-protected storage. Should this not be possible, the Adapter's configuration file does support the properties (see About Configuration). In this case, encrypting the values is strongly encouraged.

Using the Utility

The executable (CipherGen.exe) must be placed in the same directory as the Avalara.AvaTax.Common.Security.dll.


1. Double-click CipherGen.exe or type "CipherGen.exe" into a command window (where the current directory is the location of the executable). When the application is running, you will be prompted to enter the text to be encrypted:

CipherGen will not overwrite an existing output file. It defaults to "cipher.txt". If this file already exists, it will try to output the encrypted text to "cipher_1.txt". If this file already exists, it will try "cipher_2.text", and so on.


3. Open the file specified, copy the encrypted text, and paste it into the configuration file between the <Account> element tags: 

<RequestSecurity type="Avalara.AvaTax.Adapter.RequestSecurity, Avalara.AvaTax.Adapter">
    <Account>0100000...D894330C1A1D01D6FD173</Account>
    <License></License>
    <Timeout>300</Timeout>
</RequestSecurity>
The actual encrypted text in the example above has been shortened for readability purposes. The entire text should be copied, unaltered, from the cipher.txt file into the configuration file.
