The adapter contains configuration properties for setting values for URL or request timeouts and includes a subset of properties for security specific values, such as the account and license provided by Avalara to access the web services.

Configuration is a member of the primary services, AddressSvc and TaxSvc, and can be accessed after either class has been instantiated. Setting a property is accomplished using the syntax [Service].Configuration.[Property] = value. For example: 

taxSvc.Configuration.Url = "http://www.avalara.com/avatax.services/"
Likewise, security-specific properties are accessed as: [Service].Configuration.Security.[Property] = value.

STORING CONFIGURATION PROPERTIES

The AvaTax Adapter API provides full support for programmatically setting and accessing configuration properties. However, there may be cases when it is necessary or desired to persist values in a configuration text file. The configuration file also provides full support for the configuration properties. Note: though the configuration file accepts Account and License elements, it is recommended that these values be stored in a more secure location, such as a password-protected database.

CONFIGURATION FILE STRUCTURE

The configuration file must be named "Avalara.AvaTax.Adapter.dll.config". Casing is not important.

Since the configuration file is simply an XML document, it must be identified as such. The first line of the file should be: 

<?xml version="1.0" encoding="utf-8" ?>
and is followed by the root element: 
<configuration>
.
.
.
</configuration>
Between the root element tags are the ServiceConfig, RequestSecurity, and AvaLogger element blocks. Each will be covered in detail below.

ServiceConfig

This element block's children mirrors exactly the properties of the Configuration class exposed from both AddressSvc and TaxSvc.

<ServiceConfig type="Avalara.AvaTax.Adapter.ServiceConfig, Avalara.AvaTax.Adapter">
    <Url>https://www.avalara.com/avatax.services/</Url>
    <ViaUrl></ViaUrl>
    <RequestTimeout>100</RequestTimeout>
</ServiceConfig>
All XML tags must exist in the configuration file exactly as shown in the example above. Only the values between the tags, in this case https://www.avalara.com/avatax.services/ and 100, can be altered. Any one or more of the properties can be left out. For an explanation of these elements, please see the corresponding ServiceConfig class member.

RequestSecurity

This element block's children mirrors exactly the properties of the Security class exposed from both AddressSvc.Configuration and TaxSvc.Configuration. 

<RequestSecurity type="Avalara.AvaTax.Adapter.RequestSecurity, Avalara.AvaTax.Adapter">
    <Account>0100000...</Account>
    <License>0100000...</License>
</RequestSecurity>
The "..." in the Account and License text substitutes for a number of characters and/or numbers.

All XML tags must exist in the configuration file exactly as shown in the example above. Only the values between the tags, in this case 0100000..., 0100000..., and 300, can be altered. Any one or more of the properties can be left out. For an explanation of these elements, please see the corresponding RequestSecurity class member.

AvaLogger

This element describes the type of logging. 

<AvaLogger logFilePath="" logLevel="none" logMessages="false" logTransactions="false" logSoap="false">

Attribute details:

  • logFilePath: where adapter log files will be created; default is logs folder in current directory
  • logLevel: will be applied for all three types of adapter logging, default level is NONE Example for logLevel - NONE, DEBUG, INFO, WARNING, ERROR, FATAL
  • logMessages: is a switch to turn ON or OFF adapter logging; Log Filename: Adapter.2008-07-17
  • logTransactions: is a switch for logging transactions required for adapter; Log Filename: AdapterTransactions.2008-07-17
  • logSoap: is a switch for logging adapter SoapTrace; Log Filename: AdapterSoap.2008-07-17

    • EXAMPLE

    A sample configuration file can be found at SDK\Samples\Avalara.AvaTax.Adapter.dll

    The following example shows a standard configuration file setup. 

    <?xml version="1.0" encoding="utf-8" ?>
<configuration>
    <ServiceConfig type="Avalara.AvaTax.Adapter.ServiceConfig, Avalara.AvaTax.Adapter">
        <Url>https://www.avalara.com/avatax.services/</Url>
        <ViaUrl></ViaUrl>
        <RequestTimeout>100</RequestTimeout>
        <TraceSoap>true</TraceSoap>
    </ServiceConfig>
    <RequestSecurity type="Avalara.AvaTax.Adapter.RequestSecurity, Avalara.AvaTax.Adapter">
        <Account>0100000...</Account>
        <License>0100000...</License>
    </RequestSecurity>
    <AvaLogger logFilePath="" logLevel="debug" logMessages="true" logTransactions="false" logSoap="true">
</configuration>
     

    The "..." in the Account and License text substitutes for a number of characters and/or numbers.
